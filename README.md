# StackLabirint

## Условие задачи
Создать программу работы со стеком, выполняющую операции добавления, удаления элементов и вывод текущего состояния стека. Реализовать стек:

a) статическим массивом
  
b) списком

Используя операции со стеком реализовать поиск пути в лабиринте, который представлен матрицей. Все стандартные операции со стеком должны быть оформлены подпрограммами.

При реализации стека списком в вывод текущего состояния стека добавить возможность просмотра адресов текущих и освобожденных элементов стека. Оценить временную и емкостную эффективность стека при различной реализации.
## Исходные данные: 
Взаимодействие с программой происходит через меню-интерфейс. Сначала выбирается режим работы: 1 — стек на основе массива, 2 - стек на основе связанного списка.

Далее запускается основной цикл программы. Пользователь вводит числа (0-4) для выбора действий программы.

1, чтобы считать лабиринт

2, чтобы считать лабиринт из файла labirint.txt

3, чтобы вывести лабиринт на экран

4, чтобы найти путь

0, чтобы завершить

Лабиринт вводится в виде матрицы: сначала вводятся 2 числа размер матрицы (n строк и m столбцов), затем вводятся n строк по m элементов. Каждый элемент описывается числами 0, -1, 1 или 2. Числа 1 и 2 означают точки старта и конца лабиринта соответственно, оба должны присутствовать в одном и только одном экземпляре. Число 0 означает свободную клетку, а -1 - препятствие.
## Выходные данные:
При выборе пункта меню 3, матрица выводится в том же формате, что и при вводе. Если перед выводом матрицы был произведён расчёт пути, ячейки тупиковых путей будут помечены числами -2, а числа найденного пути (если путь найден) числами -3.

При выборе пункта 4, выводится набор точек (x, y) от стартовой точки (включительно) до конечной (невключительно), если путь найден, иначе выводится сообщение о том, что путь не найден. Также выводится время, затраченное на поиск пути и память на хранение стека (не учитывая значения элементов). Для стека на основе листа, также выводится список адресов памяти, использованных в ходе работы с ним.
## Задача, реализуемая программой
Поиск пути в лабиринте, с использованием стека на основе массива и списка, замеры времени и памяти на выполнение.
## Способ обращения к программе
Через терминал (app.exe).
## Возможные аварийные ситуации и ошибки пользователя
Некорректное обращение к меню. исспользование недопустимых символов, некорректные размеры матриц, описания полей. Попытка считать матрицу из несуществующего файла labirint.txt. Нехватка памяти для инициализации матрицы или стека (элемента стека), переполнение стека. Попытка поиска пути в неинициализированном лабиринте.
